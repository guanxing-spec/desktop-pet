# 桌面显眼包 — V12.0 计划分析 vs 当前项目状态

## 关键变化：V10 → V12

V12.0 计划做了**重大架构变更**：

| 维度 | V10（当前实现） | V12（新计划要求） |
|------|----------------|-------------------|
| 渲染方式 | Canvas 程序化绘制（`ctx.ellipse/arc` 画少女） | **9 层 PNG 纸娃娃系统** |
| 动画机制 | 正弦波弹性变换（`bobY/tilt/squash`） | **骨骼关节旋转 + Pose 关键帧插值** |
| 动作定义 | 数学公式驱动的弹性动画 | **3~5 个 Pose 关键帧，progress 0→1 插值** |
| 资源依赖 | 零外部资源（纯代码绘制） | **9 张 PNG + 锚点标注 + imagemin 压缩** |
| 骨架系统 | 无骨骼概念，整体变换 | **父级→子级关节树（躯干→上臂→前臂→...）** |

## 需要新做（V12 核心变更）

### Phase 1：骨骼引擎 + 纸娃娃系统（原 Day 3-5）
**这是最大的变更，完全替换当前的 Canvas 程序化绘制：**

1. **Day 3 上午** — 准备 9 张 PNG 资源
   - 拆分少女为 9 个部件图层
   - `imagemin` 压缩，总大小 < 2.5MB
   - 存放至 `public/assets/sprites/`

2. **Day 3 下午** — 骨骼树静态拼装（先彩色矩形，不上 PNG）
   - 用彩色矩形代替 PNG 验证骨骼结构
   - `petX/petY` 全局偏移量
   - 关节连接点红点标注

3. **Day 4** — lerpAngle + Pose 插值引擎
   - `lerpAngle` 防 180° 翻转
   - Pose 定义（关节角度列表）
   - `progress 0→1` 线性插值

4. **Day 5** — FSM 绑定 + 完整舞蹈
   - CrazyDance 至少 5 个关键帧
   - FSM 驱动 Pose 进度
   - 帧率 ≥ 55fps

### Phase 2：集成与发布（原 Day 9-10）
